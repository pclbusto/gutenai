¿Qué es Workspace?

Una capa de IO “limpia” sobre una carpeta EPUB ya descomprimida.
No toca el modelo (manifest/spine/nav) más que para leer; su trabajo es:

  • Resolver rutas (href ⇄ Path) a partir del OPF.
  • Leer/escribir archivos con seguridad atómica.
  • Clasificar ítems (html, css, imágenes, fuentes, etc.) a partir de su media-type, extensión y properties.
  • Hacer backups y mover/borrar/crear archivos.
  • Publicar eventos para que la UI o el modelo reaccionen (y opcionalmente vigilar cambios con watchdog).

Piezas clave

1) EventBus mínimo

  • subscribe(name, cb) / publish(name, payload).
  • Silencioso ante errores en callbacks (no revienta el flujo).

2) Namespaces & constantes EPUB

  • NS para XPath (container.xml, content.opf).
  • Directorios por defecto: OEBPS/Text, Styles, Images, Fonts.
  • Mapas de tipos: por MIME (_MEDIA_TYPE_MAP) y por extensión (_EXT_MAP).
  • Etiquetas semánticas (ITEM_IMAGE, ITEM_STYLE, etc.) alineadas a EPUB3.

3) WorkspaceInfo

  • Guarda root (carpeta del EPUB inflado), opf_path y opf_dir.

4) Ciclo de vida del workspace

  • from_epub(epub_path, out_dir): descomprime el .epub en out_dir, valida mimetype (best-effort), descubre el OPF, crea .gutenai.lock y dispara workspace.opened.
  • create_empty(out_dir, title, lang): arma un EPUB mínimo en disco:
  • mimetype, META-INF/container.xml.
  • OEBPS/content.opf con manifest + spine.
  • Text/nav.xhtml, Text/chap1.xhtml, Styles/style.css.
  • Dispara workspace.created.
  • remove(): borra todo y dispara workspace.removed.

5) OPF y rutas

  • _discover_opf(): lee META-INF/container.xml, encuentra el full-path al OPF y setea opf_path/opf_dir.
  • resolve_href(href): href → Path absoluto (relativo al OPF).
  • rel_href(path): Path → href relativo al OPF (para reescribir manifest si alguna vez lo haces).

6) Manifest y clasificación

  • iter_manifest(): parsea el OPF y devuelve lista de dicts {id, href, media_type, properties, path_abs}.
  • classify_item(item): decide el tipo semántico:
  • Primero mira properties (nav, cover-image).
  • Luego media-type.
  • Después extensión.
  • Por último heurísticas (mt.startswith("image/"), etc.).
  • Azúcares: list_documents(), list_images(), list_styles(), list_fonts(), etc.

7) IO segura

  • _coerce_path(href_or_path): acepta href o Path y lo resuelve dentro del workspace.
  • read_text / read_bytes / write_text / write_bytes / create_file / delete_file / move.
  • Escritura atómica: _atomic_write() usa archivo temporal y os.replace() (seguro en POSIX/NTFS).
  • mimetype_for(path): adivina MIME por nombre (útil para clasificar nuevos assets).
  • Publica eventos: file.created / file.changed / file.deleted / file.moved.

8) Listado y backups

  • list_all_files(): rglob de todo el árbol (FS, no OPF).
  • snapshot_zip(out_zip=None): zip de la carpeta (ideal “Guardar punto de restauración”).

9) Watchdog (opcional)

  • Si watchdog está instalado: start_watching() levanta un observer recursivo y publica eventos en cambios del FS.
  • stop_watching() lo detiene.

10) Lock simple

  • _touch_lock() crea .gutenai.lock con el pid.
  • _unlock() lo borra al limpiar.

¿Qué sí hace vs. qué no hace?

Sí:

  • Trabaja con el FS como verdad y usa el OPF solo para listar/clasificar ítems existentes.
  • Te da utilidades para editar archivos de forma segura, y enterarte por eventos.

No:

  • No edita manifest, spine o nav (eso es del modelo).
  • Si agregás un archivo al FS, el OPF no “mágicamente” lo referencia.
  • Para eso tendrías que tener otra capa (modelo) que reescriba el content.opf.

Flujo típico de uso

```python from pathlib import Path from workspace import Workspace

1) Abrir un .epub en un workspace temporal

ws = Workspace.from_epub(Path("libro.epub"), Path("tmp/ws"))

2) Inspeccionar

docs = ws.list_documents() # [(id, href, abs_path), ...] css = ws.list_styles() imgs = ws.list_images()

3) Leer/editar un CSS

style_id, style_href, style_path = css[0] css_src = ws.read_text(style_path) ws.write_text(style_path, css_src + "\n/ tweak /")

4) Crear una imagen nueva y moverla a la carpeta estándar (FS)

new_img = ws.create_file("OEBPS/Images/logo.png", b"...png bytes...")

Ojo: si querés que forme parte del EPUB “lógico”, tu capa de modelo debe

agregar al manifest del OPF y referenciarla desde los docs.

5) Backup

zip_path = ws.snapshot_zip()

6) Limpiar workspace cuando termines

ws.remove()
